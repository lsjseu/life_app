## 1. 项目背景与目标

### 1.1 项目背景

随着人们健康意识的提升，越来越多的用户开始关注个人健康管理。然而，现有的健康管理App往往功能复杂、操作繁琐，难以满足用户随时随地、便捷记录的需求。"来福"小程序以微信小程序为起点，通过语音、图像等自然交互方式，为用户提供轻量化、智能化的健康管理服务。

### 1.2 产品愿景

成为用户最贴心的个人健康伙伴，通过AI大模型技术，帮助用户建立健康的生活方式。

### 1.3 核心目标

- **短期目标（3个月）**：完成微信小程序MVP版本上线，获取1000名种子用户
- **中期目标（6个月）**：完善核心功能，用户活跃度达到30%以上
- **长期目标（12个月）**：拓展至App和Web端，建立完整的健康管理生态

### 1.4 目标用户

| 用户画像 | 特征描述 | 核心需求 |
|---------|---------|---------|
| 职场白领 | 25-45岁，工作繁忙，饮食不规律，缺乏运动 | 便捷记录饮食运动，获取个性化健康建议 |
| 慢性病患者 | 有高血压、糖尿病等慢性病史，需要长期管理健康 | 记录身体状况，监控饮食运动，获取专业指导 |
| 健康关注者 | 注重养生，希望保持健康生活方式 | 系统管理健康数据，获取科学建议 |
| 中老年用户 | 对复杂操作不熟悉，需要简单易用的产品 | 通过语音、拍照等简单方式记录健康信息 |

## 2. 功能范围与边界

### 2.1 包含范围

- 用户健康档案管理（基本信息、病历、体检报告）
- **统一记录功能（饮食、运动、健康）** - 通过自然语言对话和拍照完成
- AI健康建议与报告生成
- 语音、文字、拍照多模态交互
- 个人中心与数据展示

### 2.2 不包含范围（V1.0）

- 在线问诊功能
- 药品购买与配送
- 医疗设备直连数据采集
- 社交分享与社区功能
- 保险相关服务

## 3. 用户场景与用例

### 3.1 场景一：早餐记录

**用户**：职场白领小王

**场景**：早上上班路上，小王买了一份早餐，想记录下来

**操作流程**：
1. 小王打开"来福"小程序
2. 点击首页"记录"快捷按钮
3. 长按语音按钮，说："我今天早餐吃了一个肉包子、一杯豆浆、一个煮鸡蛋"
4. AI识别："已识别到早餐记录，包括肉包子、豆浆、煮鸡蛋，对吗？"
5. 小王："对的"
6. AI分析并记录，显示卡路里和营养成分
7. AI给出建议："早餐搭配不错，建议搭配一些蔬菜"

### 3.2 场景二：运动记录

**用户**：健身爱好者小李

**场景**：晚上跑步后，小李想记录今天的运动情况

**操作流程**：
1. 小李打开"来福"小程序
2. 点击首页"记录"快捷按钮
3. 长按语音按钮，说："我今天晚上跑步5公里，用时30分钟"
4. AI识别："已识别到运动记录，跑步5公里，30分钟，对吗？"
5. 小李："是的"
6. 系统计算消耗卡路里，评估运动量是否达标
7. 系统给出建议："今日运动量已达标，建议运动后补充水分和蛋白质"

### 3.3 场景三：健康记录

**用户**：高血压患者张阿姨

**场景**：张阿姨每天需要测量并记录血压

**操作流程**：
1. 张阿姨打开"来福"小程序
2. 点击首页"记录"快捷按钮
3. 长按语音按钮，说："我今天早上量的血压，高压135，低压85，心率72"
4. AI识别："已识别到血压记录，收缩压135mmHg，舒张压85mmHg，心率72次/分钟，对吗？"
5. 张阿姨："对的"
6. 系统显示血压趋势图，给出建议："血压控制得不错，继续保持"

### 3.4 场景四：拍照记录饮食

**用户**：大学生小陈

**场景**：小陈在食堂吃午饭，想拍下来记录

2. 点击首页"记录"快捷按钮

### 3.5 场景五：AI咨询

## 4. 功能详细描述

### 4.1 用户健康档案

#### 4.1.1 基本信息管理

**功能描述**：用户填写和管理个人基本健康信息

**界面元素**：
- 头像、昵称
- 性别、年龄、身高、体重
- 目标体重、活动水平
- 过敏史、家族病史

**业务规则**：
- 必填项：性别、年龄、身高、体重
- 身高体重用于计算BMI
- 活动水平分为：久坐、轻度、中度、重度

**权限控制**：仅用户本人可见和编辑

#### 4.1.2 病历管理

**功能描述**：用户记录和管理过往病历信息

**界面元素**：
- 病历列表（按时间倒序）
- 新增病历表单
  - 疾病名称
  - 诊断时间
  - 诊断医院
  - 病情描述
  - 治疗方案
  - 附件上传（诊断报告、处方等）

**业务规则**：
- 支持文字描述和图片上传
- 病历信息用于AI分析时参考

**权限控制**：仅用户本人可见和编辑

#### 4.1.3 体检报告管理

**功能描述**：用户上传和管理体检报告，系统自动分析

**界面元素**：
- 体检报告列表（按时间倒序）
- 上传入口（拍照/相册选择）
- 报告详情页
  - 原始报告图片
  - 结构化数据展示
  - AI分析报告
  - 健康建议

**业务规则**：
- 支持图片格式：JPG、PNG
- 系统使用OCR识别报告内容
- AI生成健康趋势分析（如有多次报告）

**权限控制**：仅用户本人可见

### 4.2 统一记录功能

#### 4.2.1 记录入口与页面架构

**功能描述**：提供统一的记录入口，通过自然语言对话和拍照完成所有记录（饮食、运动、健康）

**记录入口**：
- 首页快捷按钮：记录（大图标，突出显示）
- 我的页面：我的记录
- 浮层记录：点击首页按钮弹出对话浮层，无需跳转页面

**页面架构**：
```
统一记录对话页面
├── 对话消息列表
│   ├── AI消息（左侧，带机器人头像）
│   └── 用户消息（右侧，带用户头像）
├── 底部输入区
│   ├── 语音输入按钮（长按说话）
│   ├── 拍照按钮
│   ├── 文字输入框
│   └── 发送按钮
└── 快捷提示（常用记录方式）
    ├── "说点什么..."
    ├── "拍张照片..."
    └── "我吃了..."、"我运动了..."、"我量了..."
```

**设计理念**：
- 一个入口，多种记录方式
- 无表单，纯对话式交互
- AI自动识别记录类型（饮食/运动/健康）
- 用户只需要自然表达，AI理解并处理

#### 4.2.2 自然语言对话记录详解

**功能描述**：通过自然语言对话完成所有记录，AI自动理解意图、识别内容、确认信息

**交互流程**：
```
1. 用户点击"记录"按钮
   ↓
2. 弹出对话浮层
   - AI欢迎语："你好！想记录点什么？可以说话、拍照或打字告诉我"
   - 显示快捷提示："我吃了..."、"我运动了..."、"我量了..."
   ↓
3. 用户选择方式一：语音输入
   - 长按语音按钮
   - 显示"正在聆听..."动画 + 声波可视化
   - 用户说话（自然语言描述）
   - 实时语音转文字
   - 用户松开按钮
   ↓
   或方式二：文字输入
   - 用户在输入框输入文字
   - 支持自然语言描述
   - 点击发送
   ↓
4. AI理解并识别
   - 识别记录类型：饮食/运动/健康
   - 提取关键信息
   - 生成确认消息
   ↓
5. AI发送确认消息
   - "已识别到[记录类型]记录：[关键信息]，对吗？"
   - 提供"对的"、"不对"、"补充"快捷按钮
   ↓
6. 用户确认或调整
   - 用户说"对的"或点击"对的"按钮 → 继续
   - 用户说"不对"或点击"不对"按钮 → AI询问"哪里不对？"
   - 用户说"补充..."或点击"补充"按钮 → 继续添加信息
   ↓
7. AI分析并处理
   - 饮食：分析食物、计算卡路里和营养成分
   - 运动：分析运动类型、计算消耗卡路里
   - 健康：分析健康指标、显示状态
   ↓
8. AI展示结果
   - 显示分析结果
   - 给出建议
   - 询问"还需要记录什么吗？"
   ↓
9. 用户可以继续记录或结束
   - 继续记录 → 回到步骤3
   - 结束 → 关闭浮层，返回首页
```

**语音输入支持的表述示例**：

**饮食记录**：
- "我早餐吃了一个包子、一杯豆浆、一个鸡蛋"
- "中午吃了米饭、红烧肉和青菜"
- "晚上喝了一碗粥，吃了点小菜"
- "刚才吃了一个苹果"
- "今天喝了三杯咖啡"

**运动记录**：
- "我今天跑步30分钟"
- "下午游泳1小时"
- "晚上做了45分钟瑜伽"
- "快走了5公里"
- "力量训练40分钟"

**健康记录**：
- "我量的血压，高压135，低压85"
- "体重65.5公斤"
- "血糖5.8，空腹"
- "体温36.5度"
- "心率72次/分钟"

**AI确认示例**：

**饮食确认**：
```
用户："我早餐吃了一个肉包子、一杯豆浆、一个煮鸡蛋"

    对吗？"
用户："是的"

#### 4.2.3 拍照记录详解

**功能描述**：通过拍照完成记录，AI自动识别图片内容并确认

**交互流程**：
```
1. 用户点击"记录"按钮
   ↓
2. 弹出对话浮层
   ↓
3. 用户点击拍照按钮
   ↓
4. 弹出选择器：拍照 / 从相册选择
   ↓
5. 选择拍照
   - 调用相机
   - 支持网格线辅助构图
   - 支持闪光灯控制
   ↓
   或选择相册
   - 打开相册
   - 支持多选（最多3张）
   ↓
6. 显示预览
   - 可删除图片
   - 可重新拍照/选择
   ↓
7. 用户点击"确认"
   ↓
8. 显示"AI识别中..." + 进度条
   - 显示识别步骤：
     * 图片预处理
     * 物体检测
     * 内容识别
     * 信息提取
   ↓
9. AI识别并发送确认消息
   - "我看到了：[识别内容]，这是你的[记录类型]吗？"
   - 提供"是的"、"不是"、"补充"快捷按钮
   ↓
10. 用户确认或调整
    - 用户说"是的"或点击"是的"按钮 → 继续
    - 用户说"不是"或点击"不是"按钮 → AI询问"那是什么呢？"
    - 用户说"补充..."或点击"补充"按钮 → 继续添加信息

12. AI展示结果

    - 给出建议
    - 询问"还需要记录什么吗？"
    ↓
13. 用户可以继续记录或结束

   - 18:00-24:00：晚上好，今天辛苦了！
   - 0:00-6:00：夜深了，早点休息吧！

**统一记录对话浮层交互**：

1. **浮层打开**：
   - 点击"记录"按钮 → 从底部向上滑出对话浮层
   - 浮层占据屏幕下方2/3区域
   - 显示AI欢迎消息和快捷提示

2. **语音输入流程**：
   - 用户长按语音按钮 → 显示"正在聆听..."动画 + 声波可视化
   - 用户说话 → 实时语音转文字显示在输入框
   - 用户松开按钮 → 停止录音，自动发送消息
   - AI理解并回复

3. **拍照输入流程**：
   - 点击拍照按钮 → 弹出选择器：拍照/从相册选择
   - 选择拍照 → 调用相机，拍照后显示预览
   - 选择相册 → 打开相册，选择图片后显示预览
   - 点击"确认" → 显示"AI识别中..."加载状态
   - 识别完成 → AI发送确认消息

4. **文字输入流程**：
   - 用户在输入框输入文字 → 支持自然语言描述
   - 点击"发送"按钮 → 发送消息
   - AI理解并回复

5. **快捷提示点击**：
   - 点击"我吃了..." → 自动填入"我吃了"，光标定位在后面
   - 点击"我运动了..." → 自动填入"我运动了"
   - 点击"我量了..." → 自动填入"我量了"

6. **确认交互**：
   - AI发送确认消息后，显示快捷按钮："对的"、"不对"、"补充"
   - 点击"对的" → 自动发送"对的"
   - 点击"不对" → 自动发送"不对"
   - 点击"补充" → 自动发送"补充"

7. **浮层关闭**：
   - 点击浮层右上角关闭按钮 → 关闭浮层
   - 点击浮层外部区域 → 关闭浮层
   - 向下滑动浮层 → 关闭浮层
   - 关闭时提示"还没保存记录，确定关闭吗？"（如有未保存内容）

**核心交互逻辑**：

**健康趋势卡片交互**：
1. **趋势数据展示**：
   - 体重变化趋势：支持切换近7天/近30天视图
   - 运动达标率：点击可查看详细达标情况
   - 连续记录天数：点击可查看记录日历
   - 平均饮食评分：点击可查看评分趋势

2. **数据卡片点击**：
   - 点击任意趋势卡片 → 跳转到数据统计详情页
   - 支持左右滑动查看更多趋势指标

**档案管理区交互**：
1. **基本信息卡片**：
   - 点击卡片 → 进入基本信息详情页
   - 显示当前基本信息摘要

2. **病历管理卡片**：
   - 点击卡片 → 进入病历管理详情页
   - 显示病历数量

3. **体检报告卡片**：
   - 点击卡片 → 进入体检报告详情页
   - 显示报告数量和最新报告日期

4. **我的记录卡片**：
   - 点击卡片 → 进入我的记录详情页
   - 显示记录数量和最新记录

5. **健康目标卡片**：
   - 点击卡片 → 进入健康目标详情页
   - 显示当前目标摘要

**基本信息交互**：
1. **表单编辑**：
   - 点击任意字段 → 进入编辑模式
   - 支持实时验证（如身高范围、年龄范围）
   - 点击"保存" → 验证所有必填项，保存成功后提示

2. **BMI实时计算**：
   - 用户输入身高和体重后 → 实时计算BMI
   - 显示BMI数值和对应状态（偏瘦/正常/偏胖/肥胖）
   - 点击BMI显示区域 → 查看BMI详细说明

**病历管理交互**：
1. **新增病历**：
   - 点击"+"按钮 → 进入新增表单
   - 必填项：疾病名称、诊断时间
   - 支持上传多张图片作为附件
   - 保存后返回列表，显示新增的病历

2. **病历详情**：
   - 点击病历卡片 → 进入详情页
   - 支持编辑和删除操作
   - 附件图片支持放大查看、保存到相册

**体检报告交互**：
1. **上传流程**：
   - 点击"上传报告" → 选择拍照或相册
   - 拍照：支持裁剪，确保报告清晰
   - 上传后显示"AI分析中..."，进度条显示
   - 分析完成后自动跳转至报告详情

2. **报告对比**：
   - 如有多次体检报告 → 显示"对比上次"按钮
   - 点击对比 → 显示两次报告的指标对比
   - 高亮显示改善或恶化的指标

3. **异常指标关注**：
   - 异常指标用红色标记
   - 点击异常指标 → 显示该指标的详细解释和建议
   - 支持添加"关注指标"，在首页显示

**我的记录交互**：
1. **筛选功能**：
   - 支持按记录类型筛选
   - 支持按时间范围筛选
   - 筛选结果实时更新

2. **记录详情**：
   - 点击记录卡片 → 进入记录详情页
   - 显示完整对话历史和AI分析结果
   - 支持删除记录

**数据统计交互**：
1. **图表交互**：

   - 点击图表上的数据点 → 显示详细数据
   - 支持切换图表类型（折线图/柱状图）

2. **数据导出**：
   - 选择导出内容和时间范围

**提醒设置交互**：

### 4.5.3 全局交互设计

#### 1. 语音输入统一流程

```
用户长按语音按钮
    ↓
显示"正在聆听..."动画 + 声波可视化
    ↓
用户说话 → 实时语音转文字
    ↓
用户松开按钮
    ↓
自动发送消息
    ↓
AI理解并回复
```

#### 2. 拍照输入统一流程

```
点击拍照按钮
    ↓
弹出选择器：拍照 / 从相册选择
    ↓
拍照 → 显示预览
相册 → 选择图片 → 显示预览
    ↓
用户点击"确认"
    ↓
显示"AI识别中..." + 进度条
    ↓
识别完成 → AI发送确认消息
    ↓
用户确认 → 保存
```

#### 3. 加载状态设计

| 场景 | 加载状态 |
|-----|---------|
| 页面首次加载 | 骨架屏 |
| 数据刷新 | 顶部下拉刷新动画 |
| AI分析中 | 加载动画 + 提示文案（如"AI正在分析..."） |
| 图片上传 | 进度条 + 百分比 |
| 语音识别 | 声波动画 + "正在聆听..." |

#### 4. 错误处理与反馈

**网络错误**：
- 显示"网络连接失败，请检查网络设置"
- 提供"重试"按钮
- 保留已输入内容，重试后恢复

**AI分析失败**：
- 显示"分析失败，请稍后重试"
- 提供"重新分析"选项
- 记录失败日志，便于优化

**对话理解失败**：
- 显示"抱歉，我没听懂，能再说一遍吗？"
- 提供快捷提示，引导用户表达

## 9. 风险与应对

| 风险 | 影响 | 概率 | 应对措施 |
|-----|------|------|---------|
| AI理解准确性不足 | 高 | 中 | 选择成熟大模型，建立人工审核机制，提供纠错机制 |

| 用户活跃度低 | 中 | 中 | 设计激励机制，优化用户体验 |
| 技术扩展性不足 | 中 | 低 | 采用模块化架构，预留扩展接口 |
| 对话式交互学习成本 | 中 | 低 | 提供清晰引导，设计快捷提示，简化操作 |
