## 1. 设计系统

### 1.1 设计理念

**核心设计原则**：
- **简洁高效**：4Tab导航，功能分工明确
- **自然交互**：语音、拍照等自然交互方式优先
- **温暖关怀**：健康类产品需要传递温暖和关怀
- **对话式体验**：记录功能采用纯对话式交互，无表单

**设计关键词**：
- 清新
- 温暖
- 专业
- 易用
- 对话

## 3. 交互设计规范

### 3.1 记录生活对话交互

#### 语音输入流程

```
用户长按语音按钮
    ↓
显示"正在聆听..." + 声波可视化
    ↓
用户说话 → 实时语音转文字显示
    ↓
用户松开按钮
    ↓
自动发送消息
    ↓
AI理解并回复
```

#### 拍照输入流程

```
点击拍照按钮
    ↓
弹出选择器：拍照 / 从相册选择
    ↓
拍照 → 显示预览
相册 → 选择图片 → 显示预览
    ↓
用户点击"确认"
    ↓
显示"AI识别中..." + 进度条
    ↓
识别完成 → AI发送确认消息
    ↓
用户确认 → 保存
```

#### 确认交互流程

```
AI发送确认消息
    ↓
显示快捷按钮：对的 / 不对 / 补充
    ↓
用户点击"对的" → 继续
用户点击"不对" → AI询问"哪里不对？"
用户点击"补充" → AI等待补充信息
    ↓
AI继续处理
```

### 3.2 加载状态设计

| 场景 | 加载状态 |
|-----|---------|
| 页面首次加载 | 骨架屏 |
| 数据刷新 | 顶部下拉刷新动画 |
| AI分析中 | 加载动画 + 提示文案（如"AI正在分析..."） |
| 图片上传 | 进度条 + 百分比 |
| 语音识别 | 声波动画 + "正在聆听..." |
| AI识别中 | 进度条 + 识别步骤 |

### 3.3 错误处理与反馈

**网络错误**：
- 显示"网络连接失败，请检查网络设置"
- 提供"重试"按钮
- 保留已输入内容，重试后恢复

**AI分析失败**：
- 显示"分析失败，请稍后重试"
